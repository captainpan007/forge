SD 卡烧好了 (N08)，SSH 概念懂了 (N09)。**现在是发声盒项目的「第一次握手」时刻**。

20 分钟后你会看到 Mac 终端跳出：

```
pi@raspberrypi:~ $ _
```

**那一瞬间你才真的"拥有"了 Pi。** 之前它只是块电路板，从这一刻起它是你的小电脑。

## Why · 你为什么要学这一节

这是发声盒项目的**第一次真实成就**。前 9 节全是认知和准备，这一节你能拿到具体可见的成果：一个能远程操作的、属于你的迷你 Linux 服务器。

后面所有的硬件操作、代码部署、调试、装软件 —— 全部从这里开始。**SSH 通了 = 后面 20 节畅通无阻；SSH 不通 = 后面寸步难行。**

<Callout type="tip" icon="🎯">
  **学完这一节你能做到：** 任何时候只要 Pi 通电 + 连家里 WiFi，你在 Mac 终端敲一行命令就能进它。
</Callout>

## What · 5 步真上手

### 准备清单

- ✅ Pi 4 一块
- ✅ 烧好系统的 microSD 卡（N08 完成的）
- ✅ Pi 电源（USB-C 5V/3A 官方电源）
- ✅ 你的 Mac，连着家里 WiFi
- ✅ Pi Imager 的 ⚙ 高级设置里 SSH 已开 + WiFi 已配（关键）

⚠ 如果 N08 跳过了 ⚙ 设置，先回去重烧一次。**别在这里折腾**。

## 排错指南（90% 的人会卡的地方）

### "Could not resolve hostname raspberrypi.local"

Mac 找不到 Pi 的地址。可能原因：

1. **Pi 还没启动好** — 多等 30 秒再试
2. **WiFi 没连上** — Pi Imager ⚙ 里 WiFi 名/密码填错了。重烧 SD 卡修复。
3. **mDNS 没工作** — 极少数路由器拦截。改试 `ssh pi@<Pi 的 IP>`，IP 去路由器后台看（找名字含 raspberry 的设备）

### "Permission denied (password)"

密码错了：

1. 你记错了 N08 设的密码（最常见）—— 重烧 SD 卡，这次记笔记
2. 输的时候多按了空格 —— 重试

### "Connection refused" / 卡 30 秒后报错

SSH 服务没开：

1. N08 ⚙ 里没勾 Enable SSH —— 重烧 SD 卡
2. Pi 没启动 —— 检查电源指示灯，重新插电

### Pi 红灯都不亮

电源问题：

1. USB-C 线只能传数据不能供电（很多便宜线是这样）—— 换 Pi 官方电源
2. 电源功率不够（手机充电头大多 5V/1A）—— Pi 4 需要 **5V/3A** 才稳定

## Try · 动手验证你真懂了

**真做 1 次 SSH** 是这一节的核心练习。完成下面 5 个动作：

```bash
# 1. SSH 进 Pi
ssh pi@raspberrypi.local
# (yes / 密码)

# 2. 看你确实在 Pi 上
uname -a                        # 应该看到 Linux raspberrypi ...
hostname                        # raspberrypi
cat /proc/device-tree/model     # Raspberry Pi 4 Model B...

# 3. 看 Pi 的内存
free -h                         # 看 RAM 用量

# 4. 看 SD 卡用了多少
df -h                           # 主分区使用率

# 5. 退出
exit
```

跑通这 5 步 = 这一节满分。
