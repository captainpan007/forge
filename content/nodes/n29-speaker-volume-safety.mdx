世卫组织数据：**85dB 以上长时间暴露伤害听力**，儿童更敏感。普通 USB 喇叭满音量在 90-95dB —— 超标。

20 分钟教你 3 件事：选合适的儿童玩具喇叭、用代码硬限音量、给电池盒预留位置。**这是把发声盒从 demo 升级到玩具的安全底线。**

## Why · 你为什么要学这一节

你儿子的听力比成人脆弱 3-5 倍。**满音量发声盒可能在 1 周内造成永久性听力损伤**。这一节不只是工程问题 —— 是儿童安全问题。

20 分钟覆盖：
- 喇叭选型（功率 / 接口 / 形态）
- 软件硬限音量（设硬上限，儿子按多少次都超不过）
- 距离 + 朝向（喇叭别正对耳朵）

<Callout type="warning" icon="🔥">
  **儿童听力安全是不可逆的。** 一旦伤害无法修复。这一节 20 分钟的投入 = 你儿子未来几十年的听力保险。
</Callout>

## What · 三件事

### 第 1 件：选合适的喇叭

**❌ 不要用：**
- 大功率 PC 音箱（10W+ 输出，儿童玩具过大）
- HDMI 音频走电视（电视满音量极响）
- 蓝牙音箱（连接不稳，影响体验）

**✓ 推荐用：**

| 类型 | 优点 | 缺点 | 适合发声盒？ |
|---|---|---|---|
| **USB 小喇叭**（套件常带）| 即插即用、低功率（< 1W）、便宜 | 音质一般 | ⭐⭐⭐⭐⭐ |
| 3.5mm 小音箱 | 通用 | 需配音量旋钮 | ⭐⭐⭐ |
| **PAM8403 模块 + 小喇叭单元** | DIY 灵活、低成本 | 需焊接 | ⭐⭐⭐⭐（进阶玩家）|

**首选：USB 小喇叭 + 内部胶布固定**。免折腾 + 音量天然小。

### 第 2 件：软件硬限音量

不光设默认音量，**代码里硬性上限**：

```python
# main.py 顶部
VOLUME = 0.6                    # 默认 60%
VOLUME_MAX = 0.7                # 硬上限 70% — 即使后面想调高也不行

pygame.mixer.music.set_volume(min(VOLUME, VOLUME_MAX))
```

**进阶：自动测试音量**

第一次部署前，用手机的"分贝计" App（Decibel X 等）测：

1. 发声盒离手机 10cm
2. 按按钮播一段最响的音频
3. 看读数

**目标：< 75dB（< 谈话音量）**。超了就改小 VOLUME 直到达标。

### 第 3 件：物理摆放

喇叭朝**地板 / 桌面** —— 不直对儿子耳朵：

```
× 直立朝上                   ✓ 朝向桌面
  ┌─────┐                     ┌─────┐
  │ 🔊  │  → 耳朵            │  ●  │  → 反射到地板（衰减）
  └─────┘                     └─🔊──┘
                                ↓ 桌面/地毯
```

声波反射 + 吸收能再降 5-10dB。

或者**喇叭面朝外**（远离孩子）：

```
[发声盒]  🔊→  ← 喇叭朝远端，孩子在前面操作按钮
```

### 完整 main.py 升级（加音量保护）

```python
# main.py — 加音量安全
import pygame
from pathlib import Path
import os

# ─── 音量安全配置 ──────────────────────
VOLUME = 0.6                            # 60%
VOLUME_MAX = 0.7                        # 硬上限

# 锁定系统音量 70%（一次性，每次启动设一下）
os.system("amixer set Master 70% unmute > /dev/null 2>&1")

# ─── 初始化 ───────────────────────────
pygame.mixer.init()
pygame.mixer.music.set_volume(min(VOLUME, VOLUME_MAX))
```

**两道闸门：** 软件音量 60%（pygame）+ 系统音量 70%（amixer）= 实际 ≈ 42% 音量。给儿童刚好。
