代码跑起来报错 —— **FileNotFoundError: [Errno 2] No such file or directory: 'a.mp3'**。

## What · 路径就是文件的"地址"

**绝对路径** 从 `/`（根目录）开始：

```
/home/pi/soundbox/sounds/a.mp3   # 完整路径，从根开始
/etc/wpa_supplicant/wpa_supplicant.conf
```

**相对路径** 从你"当前所在目录"开始：

```
sounds/a.mp3                # 相对于当前目录
./sounds/a.mp3              # 同上，. = 当前目录
../sounds/a.mp3             # .. = 父目录
```

### 三个特殊符号

| 符号 | 意思 | 例子 |
|---|---|---|
| `/` | 根目录（最顶层） | `/home/pi/...` |
| `~` | 当前用户的家目录 | Pi 上 `~` = `/home/pi` |
| `.` | 当前目录 | `./a.mp3` 等同于 `a.mp3` |
| `..` | 父目录（上一级） | `../sounds/a.mp3` |

### `cd` + `pwd` 实操理解

```bash
pwd                    # /home/pi   ← 你现在在哪
ls                     # 看 home 目录里有啥
mkdir soundbox         # 创建 soundbox 文件夹
cd soundbox            # 进去
pwd                    # /home/pi/soundbox

# 现在「当前目录」是 /home/pi/soundbox
mkdir sounds
echo "test" > sounds/a.mp3      # 相对路径，文件落在 /home/pi/soundbox/sounds/
ls sounds                       # a.mp3

# 同一文件，三种引用方式：
cat sounds/a.mp3                # 相对路径
cat ./sounds/a.mp3              # 相对路径（明示）
cat /home/pi/soundbox/sounds/a.mp3   # 绝对路径
cat ~/soundbox/sounds/a.mp3     # 用 ~
# 四种全部输出 "test"
```

### Python 里：用 pathlib.Path（现代做法）

老教程会教你用 `os.path.join(...)`。**别用，那是 2010 年的写法**。现在用 `pathlib.Path`：

```python
from pathlib import Path

# 构造路径
sound_dir = Path("/home/pi/soundbox/sounds")
sound_file = sound_dir / "a.mp3"          # / 操作符拼接路径

# 检查文件存在
if sound_file.exists():
    print(f"找到了：{sound_file}")
else:
    print(f"没找到：{sound_file}")

# 拿到脚本所在目录（关键！）
script_dir = Path(__file__).parent        # __file__ 是当前 .py 文件路径
sounds = script_dir / "sounds"            # 脚本同级的 sounds 文件夹
```

**关键技巧：`Path(__file__).parent / "sounds"`** —— 让脚本无论从哪儿启动，都能找到同级的资源。

### 发声盒文件结构（推荐）

```
/home/pi/soundbox/
├── main.py              ← 主脚本
├── sounds/
│   ├── meow.mp3
│   ├── woof.mp3
│   └── moo.mp3
└── README.md
```

代码里：

```python
from pathlib import Path

SOUND_DIR = Path(__file__).parent / "sounds"
SOUNDS = list(SOUND_DIR.glob("*.mp3"))    # 自动列出所有 .mp3
print(f"找到 {len(SOUNDS)} 个音频文件")
```

## Try · 动手验证你真懂了

**SSH 进 Pi 真做一次**：

```bash
# 1. 在 home 建个测试目录
cd ~
mkdir -p path-test/sounds
echo "fake-mp3" > path-test/sounds/a.mp3

# 2. 写个小 Python 脚本测试
cat > path-test/test.py << 'EOF'
from pathlib import Path

# 方法 1：相对路径（依赖 cwd）
print("相对：", Path("sounds/a.mp3").exists())

# 方法 2：基于脚本位置（推荐）
script_dir = Path(__file__).parent
print("基于脚本：", (script_dir / "sounds/a.mp3").exists())

# 方法 3：~/ 在 Python 里不展开
print("~ 路径：", Path("~/path-test/sounds/a.mp3").exists())

# 方法 4：~/ 需要 expanduser()
print("expanduser：", Path("~/path-test/sounds/a.mp3").expanduser().exists())
EOF

# 3. 在 path-test 目录里跑
cd ~/path-test
python3 test.py
# 输出: 相对: True / 基于脚本: True / ~ 路径: False / expanduser: True

# 4. 切到 /tmp 再跑（看相对路径就死）
cd /tmp
python3 ~/path-test/test.py
# 输出: 相对: False / 基于脚本: True / ~ 路径: False / expanduser: True

# 5. 清理
rm -rf ~/path-test
```

观察"相对路径"在不同 cwd 下行为不同 = 你真懂了路径。
