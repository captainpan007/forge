<HandsOnHook scene="冒烟事故现场">

你刚把 LED 接到 Pi 的 3.3V 输出针脚上，没加电阻。线刚一接好，**LED 闪了一下，然后没了**。手摸过去 —— 烫手。再仔细一闻，有股焦塑料味。

为什么？你只接了 3.3V，又不是 220V 市电。

—— 答案在这一节末尾的"动态水管"实验里。先读，最后亲手拖滑块复现这个烧 LED 的瞬间。

## Why · 你为什么要学这一节

下一节开始你会真的接线，按钮、LED、喇叭、电机一路接到 Pi 上。**接错就是冒烟、烧 Pi、电池漏液。** 真金白银的硬件损失。

但物理书里那种"麦克斯韦方程组"是你 99% 不需要的。**实操层面只要 3 个概念就够：电压、地、电流。** 60 秒讲清楚，10 分钟练熟。

<Callout type="tip" icon="🎯">
  **学完这一节你能做到：** 看任何电路图，你能说出"哪根是供电、哪根是地、电流大概从哪流到哪"。开篇那个烧 LED 的事故，你能 30 秒内诊断出原因。
</Callout>

回想一下 <CrossLink to="n01" relation="还记得">SBC 跟 PC 的区别</CrossLink>—— Pi 跟外界打交道靠 GPIO。但 GPIO 是"金属手指" —— **手指能戳东西的前提是手指里有电流**。这一节先讲电的基本规则。

## What · 核心心智模型：电路 = 水管系统

不要去背公式。**记住"水管"这个类比就够支撑后面 20 节的实操**。

下面这个演示是这一节最重要的图 —— 拖两个滑块亲手感受 **I = V / R** 是怎么运作的：

<WaterFlow />

试试这几个组合，找找直觉：
- **V 高 + R 低** → 水流爆表（红色警告） → 这就是开篇 LED 烧的瞬间
- **V 高 + R 高** → 水流适中 → LED 正常发光
- **V 低 + R 任意** → 没什么水流 → 灯暗或不亮

### 关键洞察 1：电压必须是「两点之间」的差

水压必须有两个高度才有意义 —— 山顶到山脚。**单独一根线没有电压。**

你说"GPIO 17 是 3.3V"，**实际意思是它相对 GND 是 3.3V**。

### 关键洞察 3：电流 —— 那个烧 LED 的真凶

回到开篇事故：**为什么 3.3V 接 LED 会烧？**

水管类比：你的水池水压不算高（3.3V），但如果**把进水管口直接撑开到无限大**（电阻 = 0），单位时间冲进来的水量就是无限大。

公式：**I = V / R**（电流 = 电压 / 电阻）

**亲手算一遍。下面是发声盒上的 LED 提示灯 —— 拖滑块改任意参数，电阻值实时更新：**

<LiveBlock
  initial={{ V_source: 3.3, V_led: 2.0, I_target: 10 }}
  label="LED 限流电阻计算器"
>

  当电源电压是 <LiveParam name="V_source" min={1.5} max={5} step={0.1} unit="V" />，
  红 LED 工作电压 <LiveParam name="V_led" min={1.8} max={3.2} step={0.1} unit="V" />，
  目标电流 <LiveParam name="I_target" min={1} max={30} step={1} unit="mA" />，

  那需要的限流电阻是 <LiveDerived
    label="R"
    compute="V_source - V_led <= 0 ? 0 : (V_source - V_led) * 1000 / I_target"
    digits={0}
    unit="Ω"

  />。

  **试试拖：**
  - V_source 拖到 **5V**（从 USB 直接供电）看 R 怎么变 ——
    需要的电阻更大了
  - I_target 拖到 **25mA**（亮但接近 LED 极限）看 R ——
    电阻变小了
  - V_led 拖到 **3.0V**（蓝/白 LED）看 R ——
    电阻几乎砍半，因为电压差小了

  <strong>极端情况测试：</strong>把 V_led 拖到 ≥ V_source 看会发生什么 ——
  电压差 ≤ 0，公式失效，灯根本不会亮。

</LiveBlock>

### Pi 4 GPIO 极限（必记）

```
Pi 4 GPIO：
  - 输出电压：3.3V（不是 5V！）
  - 单针脚最大电流：16mA
  - 所有 GPIO 总和：< 50mA
```

| 你想做的事 | 能不能直接 GPIO 推 |
|---|---|
| 推一个 LED（10mA）| ✅ 能 |
| 推 3 个 LED 并联 | ⚠ 接近极限 |
| 推一个继电器（30mA）| ✅ 勉强能 |
| 推一个直流小电机（150mA）| ❌ 烧 |
| 推一个大喇叭（500mA）| ❌ 烧 |

凡是动东西（电机/大喇叭）→ 几百 mA → Pi 推不动 → 必须**外置电源 + 三极管 / MOSFET / H 桥**当中间人。这些是 <CrossLink to="n21" relation="后面会讲">N21 按钮电路</CrossLink>、<CrossLink to="n33" relation="小车篇展开">N33 H 桥</CrossLink> 的内容。

<DeepDive title="为什么 Pi 4 GPIO 是 3.3V 不是 5V？" hint="懂这个能避免大量翻车">

90 年代的 5V TTL 逻辑（早期数字电路标准）耗电高、发热大。21 世纪芯片往低功耗 + 高密度走 —— Pi 4 内部用 1.8V 核心电压，对外暴露 3.3V GPIO 是个折中：

- **3.3V** = 跟现代传感器（OLED、ADC、数字温度计）兼容
- **不是 5V** = 接 5V 信号会击穿 GPIO 内部 ESD 保护，烧针脚

但你接的 Arduino 老外设可能是 5V 信号 —— 这就需要"逻辑电平转换器"（一个小芯片把 5V 降到 3.3V 给 Pi）。

铁律：**任何 5V 信号进 Pi GPIO 之前必须先降到 3.3V，否则烧。**

</DeepDive>

### 接线前必念 3 条铁律

1. **Pi 4 GPIO 最高扛 3.3V** —— 5V 进 GPIO = 烧针脚
2. **接电路前先断电** —— 哪怕只是 5V，短路打火花会吓你一跳
3. **GND 必须共连** —— 任何外置电源的 GND 都要拉一根黑线到 Pi 的 GND
