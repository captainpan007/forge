## Why · 你为什么要学这一节

发声盒的核心是按钮 —— **儿子按下按钮，发声盒响一声**。这一节解决"按下"如何被 Pi 识别。

你会学到 3 种检测按钮的方式，每种适合不同场景：

- `is_pressed`（查询）—— 当前是不是被按着
- `wait_for_press()`（阻塞等）—— 等到被按下才往下走
- `when_pressed = func`（事件触发）—— 按下就跑一个函数 ⭐ 发声盒用这个

  **学完这一节你能做到：** 接一个按钮 + 3 行代码，按下打印 "Hello"。后面只要把 print 换成"播音频"就是发声盒了。
</Callout>

## What · 接按钮 + 三种检测方式

### 准备清单

- ✅ N16 LED 电路已工作（保留，N18 会用）
- ✅ 1 个按钮（套件里的大蘑菇按钮或随便一个 tactile 按钮）
- ✅ 2 根杜邦线

### 接线（按钮电路最简单）

```
按钮一端 ────── GPIO 17（任意 GPIO，咱用 17）
按钮另一端 ──── Pi GND

按钮 = 一个开关
- 按下：两端导通 → GPIO 17 跟 GND 相连 → 读到 LOW (0)
- 松开：两端断开 → GPIO 17 悬空 → gpiozero 用内置上拉电阻拉到 HIGH (3.3V)
```

<Callout type="info" icon="🧠">
  **gpiozero 自动处理上拉电阻。** 老教程让你外接 10kΩ 电阻 — 那是给 RPi.GPIO 的玩法。gpiozero 把这个细节封装了，你只接两根线就行。<CrossLink to="n21" relation="后面会展开">N21 详解上拉/下拉电阻原理</CrossLink>。
</Callout>

<Misconception
  myth="按钮要接 3.3V + GND 两根线"
  truth="只要接 GPIO + GND 两根线。gpiozero 自动把 GPIO 配置为'内部上拉 + Input 模式'。"
  why="按钮就是一个开关 — 没按 = 断路，按了 = 通路。gpiozero 的 Button 类默认配置 GPIO 为「pull-up Input」: 没按时通过 Pi 内部电阻拉到 3.3V，按下时按钮把 GPIO 短到 GND = LOW。你不需要外部电阻或外部 3.3V。"
/>

### 方式 1：is_pressed（查询当前状态）

```python
from gpiozero import Button
import time

button = Button(17)            # BCM 17 = 物理 11 号

# 每秒检查一次
while True:
    if button.is_pressed:
        print("按住了！")
    else:
        print("没按")
    time.sleep(1)
```

**问题：** 这是「轮询」(polling) 模式 —— Pi 一直忙着检查、CPU 占用高 + 你可能错过短按（按了 0.2 秒、轮询 1 秒发现不了）。

### 方式 2：wait_for_press（阻塞等待）

```python
from gpiozero import Button

button = Button(17)

print("等你按下...")
button.wait_for_press()        # 程序停在这一行，直到按下才继续
print("终于按了！")
```

**特点：** 简单直观，但只能等**一次**按下。继续等下一次得再调一次 wait_for_press()。

### 方式 3：when_pressed（事件回调）⭐ 发声盒用这个

```python
from gpiozero import Button
from signal import pause

button = Button(17)

def on_press():
    print("按了！")

button.when_pressed = on_press     # 注册回调
pause()                             # 永远暂停（事件驱动靠这个）
```

**特点：**
- 你的"动作"（`on_press`）跟"检测"完全解耦
- Pi 不忙 CPU，按下时由系统中断触发函数
- 这就是<CrossLink to="n18" relation="下一节">事件驱动编程</CrossLink>，发声盒终极模式

### 完整示例：按钮 → 控制 LED

把按钮跟 LED 串起来 —— 按下亮、松开灭：

```python
from gpiozero import LED, Button
from signal import pause

led = LED(17)            # ⚠ 注意：LED 跟 Button 用不同 GPIO
button = Button(18)      # 按钮换到 BCM 18

button.when_pressed = led.on    # 按下时让 LED 亮
button.when_released = led.off  # 松开时让 LED 灭

print("按住按钮 LED 就亮，松开就灭")
pause()
```

**5 行核心代码 = 一个完整的按钮-LED 联动**。N17 你脑子里要有这个模型。

### 进阶玩法

```python
button.hold_time = 2                     # 按住超过 2 秒触发 held 事件
button.when_held = some_function

button.bounce_time = 0.1                 # 软件去抖 100ms（防按一次触发两次）

button.value                              # 当前值：1 (没按) 或 0 (按了)
button.active_time                        # 按住多久了（秒）
```

<Callout type="info" icon="🧠">
  **机械按钮的抖动 (bounce)**：物理按钮按下瞬间触点会"颤抖"几毫秒，导致一次按下被检测成多次。`bounce_time=0.1` 让 gpiozero 忽略 100ms 内的重复触发。发声盒里**一定要设置 bounce_time**，否则一次按下可能播两次音频。
</Callout>

```python
from gpiozero import Button

    count += 1
    print(f"按了第 {count} 次！")

button.when_pressed = on_press

print("等你按按钮... (Ctrl+C 退出)")
pause()
```
