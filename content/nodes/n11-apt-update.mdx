## Why · 你为什么要学这一节

Linux 装软件不像 macOS 拖 .app 文件 —— 它用**包管理器**。Pi 上的包管理器叫 **apt**（Advanced Package Tool）。

apt 有一份本地的"软件清单"，告诉它哪些软件可装、版本是多少。但这份清单**会过期**，所以你装新软件前必须先**更新清单**。

这一节就教两行命令 + 一个心智模型。

<Callout type="tip" icon="🎯">
  **学完这一节你能做到：** 不假思索敲出 `sudo apt update && sudo apt upgrade -y`，并能解释每个词的作用。
</Callout>

## What · 两行命令搞定

### 一句话区分 update vs upgrade

```bash
sudo apt update      # 更新「软件清单」(metadata) — 不实际装软件
sudo apt upgrade -y  # 把已装的所有软件升级到最新版本
```

**人话翻译：**
- `update` = "去网上看看现在有哪些软件，最新版本是多少"（只更新清单）
- `upgrade` = "根据最新清单，把我现有的软件都升级"

<Misconception
  myth="apt update 就把所有软件都更新了 — 一个命令搞定"
  truth="apt update 只更新'清单' — 不动你装过的软件。要真升级软件还得 apt upgrade。两个命令配合用。"

### 实操：进 Pi 第一件事

```bash
# 你已经 ssh 进 Pi 后：

sudo apt update
# (输出一堆 "Get:1 http://..."、"Hit:..."，10-30 秒)

sudo apt upgrade -y
# (-y = 所有提示都自动 yes，不让你逐个确认)
# (5-15 分钟，看升级的包数量。第一次烧好 Pi 跑可能装 200 个包升级)
```

**`&&` 把两条命令串起来：**

```bash
sudo apt update && sudo apt upgrade -y
```

`&&` 表示前一条成功才执行下一条。一个命令搞定两步，第一次进 Pi 的肌肉记忆。

### sudo 为什么必须

包管理器在改 Pi 系统级文件（`/var/lib/apt/...`、`/usr/bin/...`），普通用户没权限写。<CrossLink to="n03" relation="还记得">N03 讲过 sudo</CrossLink> —— 临时管理员权限就在这种场景用。

不加 sudo 会报：

```
E: Could not open lock file /var/lib/apt/lists/lock - open (13: Permission denied)
```

### 装一个软件

更新完清单 / 升级完所有软件后，装新的：

```bash
sudo apt install mpg321 -y       # 装 mp3 播放器（发声盒会用）
sudo apt install python3-pip -y  # 装 Python pip（已经预装但确认下）
sudo apt install git -y          # 装 git（拉代码用）
```

模式：**`sudo apt install <软件名> -y`**

  answer="这些是 Raspberry Pi 官方的「软件源」服务器 — apt 配置里写好的 URL 列表。每个软件包的清单都从这些服务器获取。你也可以加自定义源（比如装 Docker 就要加 docker.com 的源），但默认这些够你做发声盒。"
/>

### 删软件

```bash
sudo apt remove mpg321 -y           # 删 mpg321 软件，保留配置文件
sudo apt purge mpg321 -y            # 删干净，包括配置
sudo apt autoremove -y              # 清理不再被需要的依赖
```

<Callout type="info" icon="🧠">
  **频率：** 每周 1 次 `sudo apt update && sudo apt upgrade -y` 是 Linux 用户的好习惯。Pi 上跑发声盒可能 1 个月才更新一次也无所谓（嵌入式不像服务器要求那么严）。
</Callout>

## Try · 动手验证你真懂了

**SSH 进 Pi 真做一次**：

```bash
# 1. 更新清单
sudo apt update

# 2. 升级所有已装软件
sudo apt upgrade -y

# 3. 装一个新软件（mpg321 - 发声盒会用）
sudo apt install mpg321 -y

# 4. 验证装上了
mpg321 --version
# 应该看到 "mpg321 0.3.x ..."

# 5. 清理无用依赖
sudo apt autoremove -y
```

**跑通 = 你掌握了 Linux 装软件全流程。**
