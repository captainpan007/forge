## What · 悬空电平问题

### 问题来源：GPIO 是 Input 时"听"什么

Pi 把 GPIO 配为 Input → 内部高阻抗探针等着读电压。问题是：**如果 GPIO 完全悬空（什么都不接），探针读出来的电压是什么？**

答案：**完全随机**。空气中的电磁干扰、手指靠近的电容、Pi 板上其他信号串扰 —— 都可能让 GPIO 电压在 0V 到 3.3V 之间漂浮。**Pi 把它解释成 0 还是 1 是随机的。**

这就是为什么 N17 你没接按钮去读 GPIO 17 看到值随机跳。

  truth="不接线时 GPIO 处于'悬空'(floating)状态，电压由环境干扰决定 — Pi 读到 0/1 完全随机。这是物理特性，不是 bug。"

/>

### 解决方案：上拉 / 下拉电阻

**核心思想：用电阻给 GPIO 一个'默认值'**

```
上拉（pull-up）：
    +3.3V ─── 10kΩ ─── GPIO ─── [按钮] ─── GND

    没按按钮时：GPIO 通过 10kΩ 拉到 3.3V → 读到 1
    按下按钮：  GPIO 直接接 GND → 读到 0
    （按钮短路掉电阻）

下拉（pull-down）：
    +3.3V ─── [按钮] ─── GPIO ─── 10kΩ ─── GND

    没按按钮时：GPIO 通过 10kΩ 拉到 GND → 读到 0
    按下按钮：  GPIO 直接接 3.3V → 读到 1
```

**两种接法都能解决悬空问题。** 区别只是"按下时读什么"。

### 为什么是 10kΩ？

电阻值不是随便挑的：

- **太小（如 100Ω）**：电流大、电池/电源耗电增加
- **太大（如 1MΩ）**：抗干扰能力差，容易被串扰拉跑
- **10kΩ** 是嵌入式行业默认 —— 抗干扰好 + 耗电低（3.3V / 10kΩ = 0.33mA 微弱电流）

Pi 内部上拉电阻典型 50kΩ 左右，对按钮场景完全够。

### gpiozero 的"魔法"揭秘

```python
Button(17)
```

这一行 gpiozero 默默做了：

1. 把 GPIO 17 配为 Input
2. **激活 Pi 内部上拉电阻**（默认 pull_up=True）
3. 配置中断（事件驱动用）

所以你只需要接**两根线**：GPIO + GND（按钮一端接 GPIO，另一端接 GND）。**Pi 内部已经把 GPIO 通过电阻连到 3.3V** —— 你不用外接。

**如果你想用下拉：**

```python
Button(17, pull_up=False)
```

Pi 内部改成下拉。这时候按钮接法要改成：一端接 3.3V，另一端接 GPIO。

### 大按钮（机械触点）的特殊性

大蘑菇按钮内部就是一对**金属触点**（你<CrossLink to="n22" relation="下一节会拆">N22 看到</CrossLink>），按下 → 触点闭合 → 两端短路。没按 → 触点分开 → 两端断开（无穷大电阻）。

所以**大按钮 + 上拉**：
- 没按：GPIO 通过 10kΩ 拉到 3.3V = HIGH = is_pressed 返回 False
- 按下：触点把 GPIO 短到 GND = LOW = is_pressed 返回 True

完美匹配。

## Try · 动手验证你真懂了

**真做悬空 vs 上拉对比实验**：

```bash
ssh pi@raspberrypi.local

# 实验 1：完全不接，看 GPIO 漂浮
python3
```

```python
>>> from gpiozero import DigitalInputDevice
>>> g = DigitalInputDevice(17, pull_up=None)   # pull_up=None = 不开内部上拉
>>> for _ in range(10):
...     print(g.value)
...     import time; time.sleep(0.5)
...
1
0
1
1
0
0
...                  ← 随机跳，证明悬空
>>> exit()
```

```python
# 实验 2：开内部上拉，再读 10 次
>>> g = DigitalInputDevice(17, pull_up=True)
>>> for _ in range(10):
...     print(g.value)
...     time.sleep(0.5)
...
1
1
1
1
1
1
...                  ← 全是 1，稳定
```

亲眼看到 GPIO 在悬空时的"灵异跳变" + 上拉后的稳定 = 你真懂这个问题了。
