SSH 进 Pi 了 (N10)。装好软件了 (N11)。现在你想：**改一下 Pi 的 hostname**（让它叫 "soundbox" 而不是默认的 "raspberrypi"）。

## Why · 你为什么要学这一节

Linux 配置任何东西的 95% 路径是：**改某个文件 → 重启服务**。

- 改 WiFi → 编辑 `/etc/wpa_supplicant/wpa_supplicant.conf`
- 改 hostname → 编辑 `/etc/hostname`
- 改启动脚本 → 编辑 `/etc/rc.local` 或写 systemd 文件
- 改 Python 代码 → 编辑 `~/soundbox/main.py`

**没有编辑器你寸步难行。** nano 是最低门槛的选择，5 分钟上手。

<Callout type="tip" icon="🎯">
  **学完这一节你能做到：** 在 Pi 上能打开 / 修改 / 保存 / 退出任何文本文件，包括需要 sudo 的系统级文件。
</Callout>

## What · nano 5 分钟教程

### 一句话

**nano = Linux 上最简单的命令行文本编辑器。** 跟你 Mac 上的 TextEdit 比，nano 没有鼠标 / 没有菜单 / 全靠快捷键。但所有快捷键都列在屏幕底部 —— **不用记**。

### 打开 / 关闭 nano

```bash
nano hello.txt              # 打开 hello.txt（不存在则新建）
nano /etc/hostname          # 编辑系统文件（提示无权限）
sudo nano /etc/hostname     # 用 sudo 改系统级文件
```

打开后画面：

```
  GNU nano 7.2                  /etc/hostname

raspberrypi
_

^G Help    ^O Write Out  ^W Where Is   ^K Cut    ^T Execute
^X Exit    ^R Read File  ^\ Replace    ^U Paste  ^J Justify
```

底部那两行就是**所有快捷键**。`^` 代表 **Ctrl 键**（Mac 也是 Control，不是 Cmd）。

### 4 个你需要记的快捷键

| 快捷键 | 干啥 |
|---|---|
| **Ctrl + O** | 保存（Write Out） |
| **Ctrl + X** | 退出 |
| **Ctrl + K** | 剪切当前行 |
| **Ctrl + U** | 粘贴 |

**其他都不用记。** 看屏幕底部。

### 实操：改 Pi 的 hostname

```bash
# 1. 用 sudo 打开（因为是系统文件）
sudo nano /etc/hostname

# 2. 把里面的 "raspberrypi" 改成 "soundbox"
#    （直接打字 / 删字，跟普通编辑器一样）

# 3. Ctrl+O 保存（会问文件名，直接回车）
# 4. Ctrl+X 退出

# 5. 还需要改 /etc/hosts 把 raspberrypi 也改成 soundbox
sudo nano /etc/hosts
# (找到 127.0.1.1 那行的 raspberrypi 改成 soundbox)

# 6. 重启 Pi 生效
sudo reboot

# 重启后再 ssh，提示符变成 pi@soundbox:~ $
```

### 常见操作技巧

**搜索**：`Ctrl + W` → 输关键词回车 → 跳到匹配位置

**跳到指定行**：`Ctrl + _` → 输行号回车（调试 Python 报错时巨好用）

**全选+复制**：nano 用 `Ctrl + 6` 标记起始位置，移动光标到结束位置，`Ctrl + K` 剪切（即拷贝走+删除）。这个稍麻烦 —— 通常配置文件改少量内容直接打字就行。

**取消未保存退出**：`Ctrl + X` 时如果有未保存修改，nano 会问 "Save modified buffer?" —— **`N` 表示不保存，`Y` 表示保存，按 Esc 或 Ctrl+C 取消退出**。

## Try · 动手验证你真懂了

**SSH 进 Pi 真做一次**：

```bash
# 1. 在 Pi 的 home 目录建一个文件
cd ~
nano my-first-edit.txt

# 2. 输入这一行
echo "我学会 nano 了"

# 3. Ctrl + O 保存（提示文件名时直接回车）
# 4. Ctrl + X 退出

# 5. 用 cat 验证内容确实写入了
cat my-first-edit.txt

# 6. 删掉这个测试文件
rm my-first-edit.txt
```

跑通这 6 步 = 你掌握了 nano 的核心循环：**nano 文件名 → 改 → Ctrl+O 保存 → Ctrl+X 退出**。
