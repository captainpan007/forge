发声盒最终要给儿子用：**Pi 得藏在玩具盒里，看不见、摸不着、不接显示器键盘**。

那你怎么调试？怎么改代码？怎么看错误？

—— 答案就是 **SSH**。一行命令从你 Mac 远程进 Pi，跟坐在它面前一样能敲命令、改文件、装软件。

10 分钟看完你就明白：SSH 不是什么黑科技，**就是给两台电脑之间架一条加密的"键盘+屏幕电缆"**。

发声盒做完后 Pi 藏起来 → 你必须 SSH 远程操作。

## What · SSH 一句话

**SSH = Secure SHell**。一种让你**从 A 机器远程操作 B 机器命令行**的协议，全程加密。

### 一行命令长这样

```bash
ssh pi@raspberrypi.local
```

拆开看：
- `ssh` — 命令名
- `pi` — 在 Pi 上的用户名（你烧卡时设的）
- `@raspberrypi.local` — Pi 的"网络地址"（hostname，烧卡时设的）

按回车 → 输密码 → 进 Pi。**就这么简单。**

### "网络上的人看不见我打的字吗？"

看不见。SSH 在 1995 年发明，**核心创新就是端到端加密**：

```
你 Mac 上敲: "ls -la"
        ↓
     加密
        ↓
WiFi 路由器 → 互联网 → Pi 的网卡
   (看到的是乱码)
        ↓
     解密
        ↓
Pi 收到: "ls -la"
```

即使有人在中间监听你的 WiFi 流量，截到的也是加密后的乱码。

### 密码 vs 密钥（基本概念）

**密码登录**（你烧卡设的方式）：每次 ssh 都让你输密码。

```bash
ssh pi@raspberrypi.local
pi@raspberrypi.local's password: ▮▮▮▮▮▮▮
```

**密钥登录**（v0.4 进阶节点会讲）：你生成一对密钥（公钥 + 私钥），把公钥放 Pi 上，以后 ssh 不用输密码 —— 直接进。

```bash
# 密钥配置好后：
ssh pi@raspberrypi.local
# (直接进，不问密码)
```

**为什么密钥更好：**
- 不用记密码 / 不会因为输错被锁
- 公钥可以公开放 GitHub，私钥永远不出你 Mac
- 暴力破解几乎不可能（密钥长度远超密码）

**现在 N10 用密码就够。** 密钥后面再学。
