### 代码逐行解释

```python
from gpiozero import LED       # 从 gpiozero 库导入 LED 类
led = LED(17)                  # 创建一个 LED 对象，绑定 BCM 17
led.blink()                    # 让它无限闪烁（默认 1 秒亮 1 秒灭）
input("按回车停止...")          # 阻塞等待 — 没这一行程序立刻退出，LED 也停
```

**`gpiozero.LED(17)` 背后做了什么：**
1. 告诉 Pi："把 BCM 17 配成 Output 模式"
2. 创建一个 Python 对象，让你用 `.on()` / `.off()` / `.blink()` 操作它
3. 程序退出时自动释放 GPIO（不会一直占用）

### 进阶玩法（每个一行）

```python
led.on()                       # 一直亮
led.off()                      # 灭
led.blink(on_time=0.5, off_time=0.5)        # 半秒一次（更快）
led.blink(on_time=2, off_time=0.5, n=3)     # 闪 3 次后停（n = 次数）
led.toggle()                   # 翻转当前状态
led.pulse()                    # 渐亮渐灭（需要 PWM 引脚 / GPIO 13/18 等）
```

  truth="LED 不亮 9 成是接线问题，1 成是代码问题，几乎不会是 LED 坏。"
  why="LED 是世界上最便宜耐用的元件之一 — 不超流（电阻够大），它能亮 10 年。新手第一次接电路，'灯不亮'的真实原因排序：(1) LED 长短腿插反，(2) 电阻没串进电路，(3) GND 没连 Pi，(4) GPIO 编号错（接物理 17 = 3.3V 而不是 BCM 17 = 物理 11）。"

### 排错 5 步

LED 没亮？按顺序检查：

1. **看代码跑没** —— python3 blink.py 后终端有"按回车停止..."提示吗？没就是代码错（如缩进 / 拼写）
2. **LED 长短腿** —— 长腿（+）必须接电阻那侧，短腿接 GND
3. **GPIO 编号** —— 代码 LED(17) = 物理 11 号针脚 ≠ 物理 17 号（重读 <CrossLink to="n06" relation="见">N06</CrossLink>）
4. **GND 接了吗** —— 没 GND 整个电路不工作
5. **电阻** —— 220Ω 不能省，没电阻 LED 几秒内烧
