## Why · 你为什么要学这一节

未来 3 周你会跟终端打几百次交道。但 **80% 的时间你只用 5 个命令**：`cd` `ls` `pwd` `cat` `sudo`。

很多教程一上来就丢 30 个命令给你，让你觉得"我永远也学不完"。事实是 —— 哪怕用了 5 年的工程师，平时也就这几个命令循环往复。其他的（`grep`、`find`、`awk`、`sed`）**用到了再现学**，初学者根本不需要先囤。

<Callout type="tip" icon="🎯">
  **学完这一节你能做到：** 不看教程、不上网搜，只用这 5 个命令在 Mac / Pi 上找到并查看任意一个文件的内容。
</Callout>

## What · 五个命令拆开讲

### 命令 1: `pwd` —— "我在哪？"

**Print Working Directory**（打印当前目录）。

打开 Mac 终端（⌘+空格 输 Terminal 回车），敲：

```bash
pwd
```

输出大概像 `/Users/你的用户名`。这就是你**当前所在的文件夹**——终端打开时默认在你的 Home 目录。

### 命令 2: `ls` —— "这里有什么？"

**LiSt**（列出）。

```bash
ls
```

输出当前文件夹里所有文件和子文件夹。加上 `-la` 看详情：

```bash
ls -la
```

`-l` = 长格式（显示权限、大小、修改时间）；`-a` = all（包含隐藏文件，那些以 `.` 开头的）。

<Callout type="info" icon="🧠">
  **为什么是 `-la` 不是 `-l -a`？** Unix 命令传统：多个单字母选项可以合并。你写 `-la` 跟 `-l -a` 完全等价。这是 50 年前的设计，至今全行业通用。
</Callout>

**输出长这样：**
```
drwxr-xr-x   5 alexpan  staff   160 May 10 14:30 .
drwxr-xr-x  20 alexpan  staff   640 May 10 12:00 ..
-rw-r--r--   1 alexpan  staff   234 May 10 14:25 forge.md
drwxr-xr-x   8 alexpan  staff   256 May 10 14:30 Projects
```

第一列开头的 `d` 表示**目录**（文件夹），`-` 表示**文件**。先认这个就够。

### 命令 3: `cd` —— "去那边"

**Change Directory**（切换目录）。

```bash
cd Projects        # 进入当前目录下的 Projects 文件夹
cd ..              # 回到上一级（.. = 父目录）
cd ~               # 回到 Home 目录（~ = 你自己的家）
cd /               # 跳到根目录（/ = 整个磁盘的最顶层）
cd -               # 回到上一次所在的目录（实用！）
```

**练习套路：**

```bash
pwd                # 看现在在哪
cd Projects        # 去 Projects
pwd                # 确认到了
ls                 # 看里面有啥
cd ..              # 回退
pwd                # 确认回来了
```

跟 Mac 用 Finder 鼠标点文件夹一回事，只是用打字。**等你练熟，打字其实更快**。

### 命令 4: `cat` —— "把这个文件给我看看"

**conCATenate** —— 原意"拼接"，常用于"打印文件内容"。

```bash
cat forge.md       # 把 forge.md 整个文件内容打印出来
```

文件长就刷屏到看不见。这时候用：

```bash
cat forge.md | less   # 用分页器查看，按空格翻页，按 q 退出
```

`|` 叫**管道**——把前一个命令的输出喂给下一个命令。后面会用很多次。

**Pi 调试场景：** 看 Pi 的硬件信息：
```bash
cat /proc/cpuinfo            # CPU 详情
cat /proc/meminfo            # 内存详情
cat /etc/os-release          # 当前 OS 版本
```

这些**都是普通文本文件**，Linux 把硬件信息也"装"成文件给你看。这是 Linux 哲学："Everything is a file"。

### 命令 5: `sudo` —— "我以管理员身份要做这事"

**Super User DO**（以超级用户执行）。

有些操作（装软件、改系统配置、写 GPIO）普通用户没权限做。前面加 `sudo` 临时提权：

```bash
sudo apt install python3       # 装软件，必须 sudo
sudo nano /etc/hostname        # 改系统名，必须 sudo
ls -la                         # 普通命令，不需要 sudo
```

第一次 `sudo` 会让你输密码（输的时候**屏幕上不会显示任何字符**——别慌，正常输完按回车就行）。

<Misconception
  myth="安全起见，所有命令前都加 sudo 才稳"
  truth="完全相反 — 习惯性加 sudo 是最危险的习惯之一。你需要才用，不需要就不加。"
  why="sudo 让你的命令获得'修改任何东西'的权限。如果你不假思索地用，万一打错命令（比如 `sudo rm -rf /` 那种），破坏力会无限放大。专业工程师只在确实需要写系统文件、装软件、操作硬件时才加 sudo —— 普通文件浏览、查看从来不加。"
/>

<Callout type="warning" icon="⚠">
  **`sudo` 是双刃剑。** 最毒的是 `sudo rm -rf /` —— 这一行能把整个系统文件全删光，然后你的 Pi 就开不了机了。**网上让你 `sudo rm -rf` 什么文件夹的，先看清楚再敲。**
</Callout>

## Try · 动手验证你真懂了

打开你 Mac 终端，跟着完整跑一遍这套流程（约 3 分钟）：

```bash
# 1. 我在哪？
pwd

# 2. Home 目录有什么？
ls -la ~

# 3. 进 Documents
cd ~/Documents
pwd

# 4. 列出 Documents 内容
ls

# 5. 创建一个临时文件夹（用到 mkdir，先剧透一下）
mkdir forge-test
cd forge-test
pwd

# 6. 创建一个文件并写入
echo "我学完 N03 啦" > note.txt

# 7. 用 cat 看内容
cat note.txt

# 8. 回到 Home
cd ~
pwd

# 9. 删掉这个临时文件夹（剧透 rm 命令）
rm -rf ~/Documents/forge-test
```

**这套全跑通你就会用了。** 真的。
