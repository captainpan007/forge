## Why · 你为什么要学这一节

你的发声盒需要让 Pi 知道**儿子刚刚按了按钮**。

Pi 是台电脑，CPU、内存、屏幕你都熟。但电脑要跟**外面的世界**说话——读一个按钮、点亮一个 LED、转一个电机——靠的是**那 40 根从主板上伸出来的金属针脚**。其中 26 根叫 **GPIO**。

如果不懂 GPIO 是什么，你就**永远没办法把按钮接到 Pi 上**。这一节是后面所有硬件节点（N06、N16、N17、N21、N24）的入口。

<Callout type="tip" icon="🎯">
  **学完这一节你能做到：** 看着 Pi 上的 40 根针脚，指出哪些是 GPIO、它们能做什么、为什么需要它们。
</Callout>

## What · GPIO 到底是什么

### 一句话定义

**GPIO = General Purpose Input / Output（通用输入输出）。**

就是 Pi 上的几十根**可以由你软件控制的金属针脚**——每一根你都能让它输出"高电平"（3.3V）或"低电平"（0V），或者反过来读它现在被外部接到了 3.3V 还是 0V。

### 用人话翻译

把每一根 GPIO 想象成 Pi 伸出来的一根**手指**。每根手指有两种工作模式：

<Analogy
  pairs={[
    { tech: 'Output 模式', life: '手指主动按按钮 — 给外面的电路一个信号' },
    { tech: 'Input 模式', life: '手指轻轻搭在按钮上 — 等外面的电路给它一个信号' },
  ]}
/>

**关键洞察**：同一根 GPIO 针脚，**软件一行代码就能切换**它当前是 Output 还是 Input。这就是为什么它叫 "General Purpose"——通用——同一根线啥都能干。

### 高电平 / 低电平 = 数字 0 和 1

GPIO 是**数字** IO，不是模拟。每一根针脚在任意时刻只有两种状态：

| 状态 | 电压 | 数字 | 形象 |
|---|---|---|---|
| 高电平 (HIGH) | 3.3V | 1 | "开" / 有信号 |
| 低电平 (LOW) | 0V | 0 | "关" / 没信号 |

<Callout type="warning" icon="⚠">
  **重要**：Pi 4 的 GPIO 工作电压是 **3.3V**，**不是 5V**。你要是接错了把 5V 直接灌进 GPIO，**这根针脚甚至整片 Pi 可能会烧坏**。这是为什么你要看 N06 的引脚图——分清哪些是 5V 输出、哪些是 3.3V 输出、哪些是 GPIO。
</Callout>

### 为什么有的针脚是 GPIO，有的不是

Pi 4 上一共 40 根针脚，但**只有 26 根**是 GPIO。其他的：

- 2 根是 **5V** 电源输出（红线）
- 2 根是 **3.3V** 电源输出（橙线）
- 8 根是 **GND**（地，黑线）
- 2 根预留给 **EEPROM**（你这辈子大概不会用）

所以你拿到 Pi 第一件事不是接线，是**看引脚图**——这是 N06 的内容。

### Output 模式：让 Pi 给外界一个信号

最常见用法：**点亮一个 LED**。

```python
# 这是 N16 的内容，先看个气氛
from gpiozero import LED

led = LED(17)   # 用 GPIO 17 这一根针脚，作为 Output
led.on()        # 让它输出 3.3V → LED 亮
led.off()       # 让它输出 0V    → LED 灭
```

发生了什么：
1. Python 代码告诉 Pi："请把 GPIO 17 这根针脚切成 Output 模式"。
2. 你调用 `.on()` → Pi 把 GPIO 17 拉到 3.3V → 电流通过 LED → LED 亮起来。
3. 你调用 `.off()` → Pi 把 GPIO 17 拉到 0V → 没电流 → LED 灭。

### Input 模式：让 Pi 听外界的信号

发声盒最关键的用法：**读按钮**。

```python
# 这是 N17 的内容，先看个气氛
from gpiozero import Button

button = Button(17)              # 用 GPIO 17 作为 Input
if button.is_pressed:            # 这一刻它是高电平还是低电平？
    print("按下了！")
```

发生了什么：
1. Python 代码告诉 Pi："请把 GPIO 17 切成 Input 模式"。
2. 你调用 `button.is_pressed` → Pi 去**读** GPIO 17 当前的电压。
3. 如果按钮被按下，电路接通，GPIO 17 被拉到一个固定电压（3.3V 或 0V，取决于电路怎么设计——这是 N21 的内容）。
4. Pi 检测到这个变化 → 你的代码继续执行。

<Callout type="info" icon="🧠">
  现在你已经看到了发声盒的两个核心动作的本质：**Pi 通过 GPIO 监听按钮 + 通过音频接口播放声音**。GPIO 是这一切的入口。
</Callout>

### 你现在脑子里应该有的画面

- Pi 是台小电脑，从主板伸出 40 根针脚；
- 26 根可以由你 Python 代码控制，叫 GPIO；
- 每根 GPIO 有两种模式：Output（推电压）和 Input（读电压）；
- 高电平 = 3.3V = 1，低电平 = 0V = 0；
- 你的发声盒 = Pi 用一根 GPIO 当 Input 监听按钮 + 用音频接口播放声音。
