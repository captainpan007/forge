<HandsOnHook scene="第一次 SSH 进 Pi">

烧好 SD 卡，把它插进 Pi，通电。你的 Mac 终端里敲：

```bash
ssh pi@raspberrypi.local
```

输完密码后画面定住一下，然后跳出来：

```
Linux raspberrypi 6.1.0-rpi8-rpi-v8 ...
pi@raspberrypi:~ $ _
```

你心里咯噔一下："这是啥？看着不像我熟悉的 Mac 终端。是不是装错系统了？"

—— **没装错。** 你看到的就是 Linux。读完这一节你会发现：你已经会 90% 了，因为 Mac 跟 Linux 是 Unix 表兄弟。

</HandsOnHook>

## Why · 你为什么要学这一节

你打开 Pi 第一件事，看到的不是熟悉的 macOS，是 **Linux**。

如果你心里发慌"这是啥？我从来没用过"——这一节专治。**Linux 不是另一个外星系统**——它跟 Mac 是亲兄弟（macOS 内核 Darwin 跟 Linux 都来自 Unix 家族），你已经会的 90% 终端操作直接通用。

<Callout type="tip" icon="🎯">
  **学完这一节你能做到：** 朋友问"Linux 跟 Mac/Windows 啥区别"，你能用一段话说清楚 —— 不夹生术语。
</Callout>

## What · Linux 到底是什么

### 一句话

**Linux = 一个免费、开源的操作系统 ——** 跟 macOS / Windows 是同一类东西，但**没有图形界面也能用**，所以工程师超爱。

**关键洞察：** macOS 的内核（Darwin）跟 Linux 都是从 Unix 来的 —— 所以你在 Mac 终端学的 `cd` / `ls` / `cat` 这些命令，**99% 在 Pi 的 Linux 上一字不改能用**。学 Linux 等于免费扩展你已有的 Mac 知识。

### Linux 的几个真相

**1. Linux 有几百个"发行版"**
所谓 Ubuntu、Debian、Raspberry Pi OS、Arch、CentOS …… 这些都是**基于 Linux 内核 + 不同的预装软件**组装的。可以理解成：Linux 内核 = 引擎，发行版 = 不同品牌的车。

你的 Pi 跑的是 **Raspberry Pi OS**（以前叫 Raspbian），它基于 Debian，是给树莓派优化过的。

**2. 你已经天天在用 Linux**
- 手机里 Android 的内核 = Linux
- 看的 Netflix / 用的 ChatGPT / 看的微博 —— 后面跑的服务器**几乎全部**是 Linux
- 路由器、智能电视、车载系统 —— 大多 Linux

你只是没意识到。

**3. "没有图形界面也能用"是 Linux 的超能力**
Pi 接显示器有桌面（跟 Mac 类似），但你 SSH 远程进去时只有一个**纯文本终端窗口**。这听起来反直觉，其实超高效 —— 你只敲命令、不动鼠标，Pi 在哪里你都能控制（即使它在另一个房间）。

发声盒做完后，你大概率会让 Pi 不接显示器、藏在塑料盒里 —— 这时候 Linux 的"无屏操作"就是必需品。

### 跟 Mac 终端的对比

打开你的 Mac 终端（按 ⌘+空格 输 "Terminal" 回车），敲：

```bash
ls -la ~
```

会列出你 Home 目录的所有文件。在 Pi 的 Linux 上敲**完全一样的命令**，得到完全一样的格式输出。

**你已经会 Linux 了，只是你不知道而已。**

## Try · 动手验证你真懂了

打开你 Mac 的终端，跟着敲（这些命令一会儿在 Pi 上一字不差能用）：

```bash
pwd          # 我现在在哪个文件夹？
ls           # 这个文件夹里有啥？
whoami       # 我是谁（用户名）？
date         # 现在几点？
uname        # 我跑的操作系统是啥？
```

最后一条 `uname` 在 Mac 上输出 `Darwin`，在 Pi 的 Linux 上输出 `Linux`。**这就是它们唯一肉眼可见的区别**——其他一切操作几乎相同。
