## What · 5 个最关键的概念

### 概念 1: 变量 —— 给数据起个名

```python
button_pin = 17           # 整数
volume = 0.8              # 小数
greeting = "你好儿子"     # 字符串
is_playing = False        # 布尔（True / False）
sound_files = ["a.mp3", "b.mp3", "c.mp3"]  # 列表
```

**没有类型声明** —— Python 自动猜。你只需要 `名字 = 值`。

<Misconception
  myth="Python 不写类型 = 不严谨 / 不专业"
  truth="Python 是「动态类型」语言 — 变量类型由值决定，运行时检查。这是设计选择，让代码短而易读。"
  why="C / Java 这种「静态类型」要写 `int x = 17;`，多打字、编译期检查。Python 选择「写 `x = 17` 即可，运行时再说」 — 牺牲一点编译期安全，换巨大的可读性和开发速度。脚本场景 + 嵌入式胶水代码完美匹配 Python 的取舍。"
/>

### 概念 2: print —— 让 Python 说话

```python
print("Hello")              # 输出 "Hello"
print("Volume:", volume)    # 输出 "Volume: 0.8"
print(f"Pin {button_pin} is high")  # f-string，输出 "Pin 17 is high"
```

**f-string**（前缀 `f"..."`）能在字符串里直接嵌变量，发声盒里**大量使用**：

```python
print(f"Playing {sound_files[0]} on pin {button_pin}")
# 输出 "Playing a.mp3 on pin 17"
```

### 概念 3: if / else —— 条件判断

```python
if volume > 0.5:
    print("声音有点大")
elif volume > 0.2:
    print("声音正常")
else:
    print("声音太小听不见")
```

**关键：缩进就是代码块**（Python 没有 `{ }`）。习惯用 **4 个空格**缩进。

VS Code 设置文件编辑会自动帮你处理 —— 你按 Tab，VS Code 自动转 4 空格。

### 概念 4: for 循环 —— 重复做一件事

```python
# 把列表里每个元素打印一遍
for sound in sound_files:
    print(f"Found: {sound}")

# 输出:
# Found: a.mp3
# Found: b.mp3
# Found: c.mp3
```

```python
# 重复 N 次
for i in range(3):       # i 从 0 到 2
    print(f"Round {i}")

# 输出:
# Round 0
# Round 1
# Round 2
```

发声盒里你会用 `for` 来：扫描多个按钮、检查多个文件是否存在、初始化多个 LED 等。

### 概念 5: def —— 定义一个"动作"（函数）

```python
def play_sound(filename, volume=0.8):
    print(f"开始播 {filename}，音量 {volume}")
    # ... 这里写真正播放的逻辑 ...
    print("播放结束")

# 调用：
play_sound("a.mp3")              # 用默认音量 0.8
play_sound("b.mp3", volume=0.5)  # 自定义音量
```

**def = define（定义）。** 函数让你把一段逻辑命名 + 复用。发声盒的核心逻辑会写成几个函数：`play_sound()` `read_button()` `pick_random()`。

## 完整示例：发声盒核心逻辑骨架

```python
# 把上面 5 个概念串起来 —— 发声盒的简化版

import random   # 标准库，下一节 N14 详解

# 配置
button_pin = 17
sound_files = ["a.mp3", "b.mp3", "c.mp3", "d.mp3"]

# 定义两个动作
def pick_random_sound():
    return random.choice(sound_files)

def play(filename):
    print(f"播放 {filename}（这里假装播了，真播放看 N19）")

# 主循环
def main():
    print("发声盒启动")
    while True:                          # 永远循环
        # 在 N17 这里会换成真按钮检测
        input("按回车模拟按钮：")        # 临时占位
        sound = pick_random_sound()
        play(sound)

# 入口
if __name__ == "__main__":
    main()
```

**11 行真代码 + 注释 = 整个发声盒的骨架。** 后面 N14-N20 把每个 `# 占位` 替换成真实现。

<Callout type="info" icon="🧠">
  **真相：** Python 项目里 80% 的代码长得就这样 —— 一堆函数定义 + 一个 main() 入口。掌握这个模式，你能读懂 90% 的小项目。
</Callout>
