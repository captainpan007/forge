## Why · 你为什么要学这一节

发声盒交付的最低门槛：**儿子按按钮就响 — 爸爸不在旁边也能玩**。

要满足这条 = main.py 必须自动启动 + 崩溃自愈 + 不依赖 SSH 会话。**systemd** 就是 Linux 的"程序总管家" — 几乎所有现代 Linux 服务（包括 Pi 上的 SSH、网络、蓝牙）都靠它管理。

## What · systemd 服务文件 3 步搞定

### 步骤 1：写服务定义文件

```bash
ssh pi@raspberrypi.local

# 用 sudo nano 写到系统目录
sudo nano /etc/systemd/system/soundbox.service
```

粘贴：

```ini
[Unit]
Description=Soundbox — dad-made toy for son
After=multi-user.target sound.target

[Service]
Type=simple
ExecStart=/usr/bin/python3 /home/pi/soundbox/main.py
WorkingDirectory=/home/pi/soundbox
User=pi
Group=pi
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

Ctrl+O 保存，Ctrl+X 退出。

### 步骤 2：启用 + 启动

```bash
# 让 systemd 读新文件
sudo systemctl daemon-reload

# 设为开机自启
sudo systemctl enable soundbox

# 立刻启动（不用等重启）
sudo systemctl start soundbox

# 查状态
sudo systemctl status soundbox
```

`status` 应该看到：

```
● soundbox.service - Soundbox — dad-made toy for son
     Loaded: loaded (/etc/systemd/system/soundbox.service; enabled)
     Active: active (running) since ...
   Main PID: 1234 (python3)
        ...
```

**绿色 `active (running)`** = 服务在跑。按按钮 — 喇叭响。

### 步骤 3：重启 Pi 验证

```bash
sudo reboot

# 等 1 分钟 Pi 重新启动
# 然后给儿子按按钮 — 应该立刻响（不用 SSH）

# 如果你想 SSH 看状态：
ssh pi@raspberrypi.local
sudo systemctl status soundbox
```

**从此 Pi 通电后 60 秒内自动开始响应按钮**。

### service 文件每行讲解

```ini
[Unit]
Description=...                # 给人看的描述
After=multi-user.target sound.target
# 等系统进入多用户模式 + 音频子系统就绪后再启动
# 否则 pygame.mixer.init() 可能失败

[Service]
Type=simple                    # 启动后保持前台运行（不 fork）
ExecStart=/usr/bin/python3 /home/pi/soundbox/main.py
# 真正的启动命令 — 必须绝对路径！
WorkingDirectory=/home/pi/soundbox
# main.py 的工作目录 — Path(__file__).parent 用这个
User=pi
Group=pi
# 不要用 root — 用普通用户 pi 跑（更安全）
Restart=on-failure
RestartSec=5
# 程序崩了 5 秒后自动重启 — 关键！main.py 有 bug 也能持续服务
StandardOutput=journal
StandardError=journal
# 输出收集到 journalctl 系统日志，方便排错

[Install]
WantedBy=multi-user.target
# 开机进多用户模式时自启动
```

<Misconception
  myth="把 python3 main.py 加到 .bashrc 就开机自启了"
  truth=".bashrc 是 'SSH 登录时跑'，不是 'Pi 启动时跑'。Pi 启动后没人登录 → .bashrc 永远不执行。systemd 才是 Linux 真正的开机自启机制。"
  why="Linux 启动流程：内核 → systemd → 一堆 .service（按依赖图启动）→ 多用户态就绪。你要让程序在'多用户态就绪'后启动，只能写 .service 文件给 systemd。.bashrc / .profile 是 shell 用户登录时的脚本，跟开机自启无关。"
/>

### journal 日志：调试服务

服务在后台跑，print 的内容去哪了？**进 journalctl 日志**：

```bash
# 实时看 soundbox 日志（按 Ctrl+C 退出）
sudo journalctl -u soundbox -f

# 看最近 50 行
sudo journalctl -u soundbox -n 50

# 看今天的所有
sudo journalctl -u soundbox --since today
```

按按钮时应该实时看到 `▶ voice-dad-hello.mp3 (11 剩余)` 这种输出。

### 常用 systemctl 命令

```bash
sudo systemctl status soundbox       # 看状态
sudo systemctl start soundbox        # 启动
sudo systemctl stop soundbox         # 停止（儿子吵到邻居用）
sudo systemctl restart soundbox      # 重启（改了 main.py 后用）
sudo systemctl enable soundbox       # 开机自启
sudo systemctl disable soundbox      # 关闭开机自启
sudo systemctl daemon-reload         # 改了 .service 文件后必须跑
```

**改了 main.py 后**：

```bash
nano ~/soundbox/main.py
# 改完 Ctrl+O Ctrl+X

sudo systemctl restart soundbox       # 让新代码生效
sudo journalctl -u soundbox -f         # 看是否正常
```

### 进阶：自动从 Git 拉新代码
